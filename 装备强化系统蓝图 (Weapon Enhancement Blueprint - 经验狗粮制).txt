1. 数据层改造 (Data Structure)
为了让物品和武器都能作为“经验包”，我们需要在底层定义它们提供的经验值，并为实体装备增加经验槽。

ItemData 升级：

新增 public int feedExpValue; (作为强化材料时提供的基础经验值)。

例如：初级打磨石提供 100 点，生锈的铁剑提供 20 点。

RuntimeEquipment 升级：

现有 level (等级) 字段外，新增 public int currentExp; (当前经验值)。

新增获取升级所需经验的逻辑（如 GetExpToNextLevel()）。可以简单设定一个通用公式（例如：所需经验 = 当前等级 * 100 * 品质系数）。

经验继承逻辑（可选）： 如果作为材料的武器本身也被强化过（比如 +5 的铁剑），它提供的经验值 = 基础 feedExpValue + 累计已消耗经验的 80%。

2. 核心逻辑层 (EnhanceManager 或整合入 ForgeManager)
处理“吃狗粮”和“升级”的物理运算。

强化结算接口 FeedMaterials(RuntimeEquipment target, List<InventorySlot> materials)：

遍历所有被选中的材料，计算总计提供的 EXP。

增加目标武器的 currentExp，并用 while 循环判定是否满足升级条件（支持一次吃满连升多级）。

扣除强化所需的金币（通常按消耗的狗粮数量或提供的经验量收费，做金币回收）。

调用 InventoryManager.RemoveItem 销毁这些狗粮。

一键放入算法 AutoSelectMaterials()：

优先级排序： 优先消耗专用的强化材料（打磨石），其次消耗普通（白）品质的无用武器，最后是稀有（蓝）品质武器。

安全锁： 绝对排除已穿戴在角色身上的装备、史诗（紫）及以上品质的装备、以及玩家手动加锁的装备（后续可做“锁定”功能）。

装填限制： 计算升到下一级还差多少经验，一旦选中的狗粮经验达到或刚好超过该阈值，停止自动装填。

3. UI 交互层 (UI_EnhancePanel)
建立独立的强化面板界面，直观展示进度与属性变化。

左侧 / 顶部：目标装备区

展示目标武器的图标、当前等级（Lv.1）、下一级（Lv.2）。

经验条 (Slider)： 直观显示进度，例如 45 / 100。如果是选中了狗粮处于预览状态，经验条可以用高亮颜色显示“预计增加的经验”。

属性变化预览： 如 攻击力：20  ->  22（绿色字体）。

右侧 / 底部：狗粮投喂区

提供 6~8 个空的狗粮槽位（像阵型沙盘一样）。

玩家点击空槽位，可以打开一个简化版的背包面板手动挑选要喂的装备/材料。

【一键放入】按钮： 点击后，按上述算法自动填满槽位，或填到刚好能升一级的量。

【确认强化】按钮： 显示本次强化需要消耗的金币，点击后播放强化特效并结算。
