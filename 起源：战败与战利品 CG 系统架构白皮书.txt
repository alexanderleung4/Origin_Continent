====================================================================
# 📂 附加档案：《起源》战败与战利品 CG 系统架构白皮书 (Defeat & Loot CG Architecture)
====================================================================

## 1. 核心定位 (Core Concept)
本系统非单纯的 Game Over 惩罚，而是融合了“叙事演出”、“战利品收集”与“生存博弈”的双向反馈系统。
* **我方战败（惩罚向）：** 展现角色受难/战损，推进剧情，施加生存惩罚。
* **敌方战败（奖励向）：** 击败特定敌方角色（如敌方女干部）作为“特殊战利品”触发福利/收服 CG。

## 2. 击杀溯源与数据映射 (Killer-Victim Tracking & Data)
放弃单一的“死亡标志”，引入精准的击杀溯源机制。
* **底层溯源 (RuntimeCharacter.cs):** 增设 `string lastKillerID`。当 HP 扣至 0 时，精准记录给出致命一击的单位 ID。
* **条件触发字典 (CharacterData.cs):** 每个角色身上的 CG 配置重构为列表/字典结构：
  - 例：`KillerID == "Monster_Goblin"` -> 触发 `CG_Goblin_Event_01`
  - 例：`KillerID == "Monster_Orc"` -> 触发 `CG_Orc_Event_01`
  - 例：`Default` (默认击杀) -> 触发 `CG_Generic_Defeat`

## 3. 结算列队与路由劫持 (The Settlement Queue & Router)
为了完美适配多单位同时阵亡（AOE击杀）的极端情况，重构战斗末尾的结算管线。
* **拦截点 (BattleManager.cs):** 战斗结束时不立刻弹结算面板，先进入“演出盘点阶段”。
* **演出队列 (CG Queue):** 遍历全场死者（先敌方后我方），将满足条件的 CG Event ID 压入 `List<string> cgQueue`。
* **串行播放 (Coroutine):** 呼叫 DialogueManager/CGManager 依次播放队列中的 CG，等待玩家点击结束后再播下一个，彻底杜绝 UI 互顶。
* **回归主线:** 队列清空后，再掉落常规金币/经验，并弹出战斗胜利/失败 UI。

## 4. 动态差分渲染与爆衣联动 (Dynamic Differential Rendering)
战败 CG 必须反映战斗中的物理损耗，与装备系统的 `Body` 槽位深度绑定。
* **耐久度读取:** 演出前读取受害者 `Body` 装备的当前 `maxDurability`。
* **三段式差分表现:**
  - 状态 A (耐久 > 50%): 轻伤/战败。立绘大致完好，表情不甘。
  - 状态 B (耐久 > 0): 半破/战损。衣服大面积破损，表情痛苦。
  - 状态 C (耐久 == 0): 全破/沦陷。最终的 R18 战败表现。

## 5. 生存惩罚与队伍解耦 (Survival Penalty & Party Decoupling)
建立多梯度的硬核生存反馈循环。
* **单体战败 (Teammate Defeated):** 个人 CG 播放完毕后，进入“虚弱状态”（HP=1，衣服耐久清零）。只要队伍中还有存活者，战斗继续，战后可带回“家”休养。
* **全军覆没 (Party Wipe):** 队伍最后一人倒下，触发全局惩罚。不直接读档，而是将玩家传送回“家”或特定“地牢”场景。剥夺一半金币，同时恶化玩家身上的“黑死咒”层数（扣除最大寿命上限）。

## 6. 全局回想画廊 (Global Gallery Archive)
保障玩家收集体验，隔离单局游戏存档污染。
* **数据隔离:** 独立于玩家的游戏进度建立 `GlobalSaveData.json`。
* **机制:** 任意存档只要触发过一次特定的 CG Event ID，即永久写入全局记录。玩家可在主界面的“回想房间 (Gallery)”中随时调阅已解锁的多差分 CG。

## 7. 动态耐久度与实体武装协议 (Dynamic Durability & Armed Entity Protocol)
本协议旨在规范战斗中装备的物理损耗、属性联动以及敌我同源的武装逻辑。

* **绝对隔离原则 (Runtime Instancing):** 严禁直接修改 `EquipmentData.maxDurability`。所有实体的装备实时耐久度，必须被隔离并储存在 `RuntimeCharacter` 的私有字典（或等效数据结构）中。怪物死亡后重刷，将自动生成带有全新耐久度的 Runtime 实例。
* **不可破坏豁免 (Indestructible Rule):** 当装备配置的 `maxDurability <= 0` 时，系统判定该装备不受磨损机制影响，禁止对其进行扣除计算，且战败时不触发“防具损毁”演播。
* **归零销毁机制 (Zero-Durability Destruction):** 战斗内（或战败结算时），当某槽位（如 Body）耐久度降至 0 时，触发强制卸下逻辑（Unequip）。失去该装备的所有属性加成，并触发 UI_SystemToast 全局播报：“[角色名] 的 [装备名] 已彻底损毁！”。
* **战术破甲 (Tactical Break):** 允许特殊技能直接削减敌方实体的装备耐久度，将“爆衣/破甲”从单纯的战败演出，升维为战斗中的核心削弱策略。
